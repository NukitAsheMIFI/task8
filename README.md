Task 8: Обработка сигналов SIGINT при умножении матриц с задержкой

Описание

Данный проект демонстрирует работу с сигналами на языке C с использованием signal и sigaction.

Программа перемножает квадратные матрицы размером 100x100 с 0.5-секундной задержкой между каждой итерацией умножения. При получении первого сигнала SIGINT (Ctrl+C) программа выводит текущие итераторы строки и столбца матриц и восстанавливает реакцию на SIGINT на реакцию по умолчанию (SIG_DFL). При получении второго сигнала SIGINT программа завершается.

Параметры командной строки

Один обязательный аргумент, определяющий метод установки реакции на сигнал:
- --signal (использовать signal)
- --sigaction (использовать sigaction)

Входные данные

Ввод в стандартный поток отсутствует.

Выходные данные

Программа выводит текущие итераторы при получении первого SIGINT:
- строка и столбец первой матрицы (A)
- строка и столбец второй матрицы (B)
- строка и столбец результирующей матрицы (C)

После завершения умножения программа выводит результирующую матрицу 100x100.

Пример запуска

# ./task8 --signal

<нажатие Ctrl+C>

>1 10
>10 1
>1 1

<повторное нажатие Ctrl+C>

# (программа завершается)

Функциональные требования

Умножение матриц: программа перемножает две квадратные матрицы размером 100x100.

Заполнение матриц: матрицы A и B заполняются случайными целыми числами от 0 до 9.

Задержка: между каждой итерацией умножения программа делает паузу 0.5 секунды (500000 микросекунд) с помощью usleep.

Первый сигнал SIGINT: при получении программа выводит текущие значения итераторов (i, j, k) для всех трёх матриц, после чего восстанавливает стандартную реакцию на SIGINT (SIG_DFL).

Второй сигнал SIGINT: после восстановления стандартной реакции повторное нажатие Ctrl+C завершает программу.

Два метода установки обработчика:
- через signal (при аргументе --signal)
- через sigaction (при аргументе --sigaction)

Стек технологий

Язык программирования: C.

Библиотеки: stdio.h, stdlib.h, time.h, unistd.h, signal.h, string.h, s21_matrix.h (собственная библиотека для работы с матрицами).

Система сборки: GNU Make.

Компилятор: GCC.

Инструкция по сборке и запуску

Сборка проекта

make

Запуск программы с signal

./task8 --signal

Запуск программы с sigaction

Очистка временных файлов

make clean

Технические подробности реализации

Временная задержка: используется функция usleep(500000) для создания задержки 0.5 секунды между итерациями умножения. Это позволяет успеть нажать Ctrl+C в процессе выполнения.

Глобальный флаг: переменная volatile sig_atomic_t got_sigint используется для безопасной передачи информации о получении сигнала между обработчиком сигнала и основным кодом. Тип sig_atomic_t гарантирует атомарность операций чтения и записи.

Обработчик сигнала: функция my_handler устанавливает флаг got_sigint = 1 при получении SIGINT и больше ничего не делает. Основная логика обработки сигнала находится в цикле умножения.

Разделение методов: функция interruptor выбирает между signal и sigaction в зависимости от переданного параметра use_sa.

Структура sigaction: при использовании sigaction заполняется структура:
- sa_handler: указатель на функцию-обработчик
- sa_mask: маска сигналов, блокируемых во время работы обработчика (очищается через sigemptyset)
- sa_flags: флаги (обнуляются)

Восстановление реакции: после получения первого SIGINT программа вызывает interruptor с SIG_DFL, что восстанавливает стандартное поведение для SIGINT (завершение программы).

Вывод итераторов: при обнаружении флага got_sigint программа выводит три строки:
- первая: i k (итераторы первой матрицы)
- вторая: k j (итераторы второй матрицы)
- третья: i j (итераторы результирующей матрицы)

Завершение программы: после восстановления стандартного обработчика повторное нажатие Ctrl+C завершает программу обычным образом.

Отличие signal от sigaction:
- signal: более простой, но менее надёжный (поведение может различаться на разных системах)
- sigaction: более сложный, но стандартизированный и предсказуемый на всех POSIX-системах
- sigaction позволяет точно контролировать маску сигналов и флаги

Примечание о библиотеке s21_matrix.h

В коде используется сторонняя библиотека s21_matrix.h. В реальном проекте необходимо:
- либо включить реализацию этой библиотеки в проект
- либо заменить вызовы функций (s21_create_matrix, s21_remove_matrix, s21_mult_matrix) на собственные реализации
- либо закомментировать s21_mult_matrix, так как в представленном коде умножение реализовано вручную в цикле
